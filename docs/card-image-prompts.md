# Card Image Prompts

Asset target directory: `public/images/cards/`

Default generation style is **editorial-photo**: realistic editorial lifestyle photography with a candid documentary feel, atmospheric scene, natural imperfect lighting, subtle film grain, and playlist-cover composition. The script appends the shared editorial-photo safety/style prompt automatically.

Keep each image scene-specific and readable at card size:

- communicate the vibe through environment, lighting, and objects
- prefer no people, back-facing people, silhouettes, cropped figures, or motion blur
- avoid close-up faces, detailed hands, fake influencer/model portraits, and celebrity resemblance
- avoid copyrighted album art, artist imagery, text, logos, and watermarks
- avoid glossy AI stock-photo perfection

Editorial-photo base prompt added by `npm run generate:card-images`:

`realistic editorial lifestyle photograph for a music discovery app, candid documentary feel, atmospheric scene, natural imperfect lighting, subtle film grain, playlist-cover composition, no text, no logos, no watermark, no celebrity, no album art, no close-up face, no distorted hands, no extra fingers, not overly polished, not glossy AI stock photo`

## Notes

- Each card should normally use `/images/cards/{card-id}.jpg`; shared filenames are deprecated unless an intentional `--dedupe` run is needed.
- Legacy `/vibes/*.png` assets remain as temporary fallbacks while new JPGs are generated.
- Use `--style editorial-photo --force` to replace older mascot/cartoon JPGs with editorial photo scenes.
- Reject images with fake-looking faces, weird hands, extra limbs, text, logos, watermarks, celebrity resemblance, album art, or glossy AI stock-photo polish.
- Delete bad JPGs and rerun `npm run generate:card-images -- --style editorial-photo --only card-id-or-output-slug --force`.
- `--style elephant` is deprecated/experimental and should not be used for default SoundLife card art.
